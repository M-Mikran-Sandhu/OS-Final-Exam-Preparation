Chapter 1: Process Synchronization

Process synchronization is about making sure multiple programs don't mess up shared data at the same time. Think of it like a bathroom - only one person can use it at a time!

1.1 Mutex (Mutual Exclusion)

What is it? A mutex is a lock that only one thread can hold at a time.

Real-world example: Imagine a printer in an office. Only one person can print at a time. If Person A is printing, Person B has to wait.

Simple Example: 
Thread A: Lock printer → Print document → Unlock printer
Thread B: Waits... Waits... Lock printer (now available) → Print document → Unlock printer

1.2 Semaphores

What is it? A semaphore is like a counter that controls how many threads can use something.

Real-world example: A parking lot with 5 spaces. When someone enters, the count goes down (5→4). When someone leaves, it goes up (4→5).

Two Types:
  • Binary Semaphore: Can only be 0 or 1 (like a mutex)
  • Counting Semaphore: Can be any number (like parking spaces)

1.3 Conditional Variables

What is it? A way for threads to wait for a specific condition to happen.

Real-world example: Waiting in a restaurant. You put your name on a list and wait. When your table is ready, the host calls your name.

How it works: 
Thread 1: Wait for condition "table is ready"
Thread 2: Table becomes ready → Signal to Thread 1
Thread 1: Wakes up and gets seated

1.4 Producer-Consumer Problem

What is it? One thread makes data (producer), another uses it (consumer).

Real-world example: A bakery. The baker (producer) makes bread. The customer (consumer) buys it. If the baker is too fast, bread piles up. If too slow, customers wait.

The Problem:
  • Buffer is full → Producer must wait
  • Buffer is empty → Consumer must wait

1.5 Bounded-Buffer Problem

What is it? Same as producer-consumer but with a limited buffer size.

Real-world example: A YouTube video buffer. It can only hold so much data. If you're downloading faster than watching, it fills up.

1.6 Reader-Writer Problem

What is it? Multiple readers can read at the same time, but only one writer can write.

Real-world example: A library. Many people can read books at the same time, but only the librarian can update the catalog.

Rules:
  • Multiple readers: ✓ Allowed
  • One writer: ✓ Allowed
  • Reader + Writer: ✗ Not allowed

1.7 Dining-Philosophers Problem

What is it? A classic problem about deadlock and resource sharing.

The Setup: 5 philosophers sit at a round table. Between each pair is one chopstick. To eat, a philosopher needs BOTH chopsticks.

The Problem: 
Philosopher 1: Takes left chopstick, waits for right
Philosopher 2: Takes left chopstick, waits for right
Philosopher 3: Takes left chopstick, waits for right
... Everyone is stuck! No one can eat!

1.8 H2O Problem

What is it? Synchronizing threads to form water molecules (H₂O).

The Challenge: You need 2 Hydrogen atoms and 1 Oxygen atom to make water. Synchronize them so they meet at the right time.

Chapter 2: Deadlocks

A deadlock happens when processes get stuck waiting for each other forever. Like two cars facing each other on a narrow road - neither can move!

2.1 What is a Deadlock?

Simple Definition: Two or more processes are stuck because each is waiting for something the other has.

Real-world example: 
Process A: Has Resource 1, wants Resource 2
Process B: Has Resource 2, wants Resource 1
Result: Both stuck forever!

2.2 Four Conditions for Deadlock

ALL FOUR must be true for deadlock to happen:

  1. Mutual Exclusion: Only one process can use a resource at a time
  2. Hold and Wait: Process holds a resource while waiting for another
  3. No Preemption: Can't take a resource away from a process
  4. Circular Wait: Processes form a circle waiting for each other

2.3 How to Prevent Deadlock

Break one of the four conditions:

  • Break Mutual Exclusion: Let multiple processes share resources (not always possible)
  • Break Hold and Wait: Request all resources at once before starting
  • Break No Preemption: Allow taking resources away from processes
  • Break Circular Wait: Number resources and always request in order

2.4 Deadlock Avoidance

What is it? Before giving a resource, check if it's safe. If it might cause deadlock, say "no".

Tool: Banker's Algorithm - like a bank checking if a loan is safe

2.5 Deadlock Detection

What is it? Let deadlock happen, then detect it and fix it.

How: Check if processes form a waiting circle. If yes, deadlock detected!

Chapter 3: Memory Management

Memory management is about organizing computer memory so programs can use it efficiently. Like organizing shelves in a library!

3.1 Contiguous Memory Allocation

What is it? Each program gets one continuous block of memory.

Real-world example: Assigning apartments in a building. Person A gets apartments 1-5, Person B gets apartments 6-10.

Pros: Simple and fast Cons: Wastes space (fragmentation)

3.2 Segmentation

What is it? Memory divided into logical pieces (code, data, stack).

Real-world example: A house with rooms. Living room, bedroom, kitchen - each serves a purpose.

Benefits:
  • Makes sense logically
  • Easy to protect different parts
  • Can share segments between programs

3.3 Paging

What is it? Memory divided into fixed-size pages (usually 4KB).

Real-world example: A notebook with pages. Each page is the same size. You can use pages in any order.

Key Idea: 
Logical Address = Page Number + Offset
Physical Address = Frame Number + Offset

3.4 Paged-Segmentation

What is it? Combines paging and segmentation. Segments are divided into pages.

Best of both worlds: Logical organization of segmentation + efficiency of paging

3.5 Fragmentation

External Fragmentation: Free memory scattered everywhere. Total free space exists, but it's not continuous.

Example: 
Memory: [Used][Free 2KB][Used][Free 3KB][Used][Free 1KB]
Total free: 6KB, but can't fit a 5KB program!

Internal Fragmentation: Allocated memory is bigger than needed. Wasted space inside the allocation.

Chapter 4: Virtual Memory

Virtual memory lets programs use more memory than physically available. The OS swaps data between RAM and disk automatically!

4.1 What is Virtual Memory?

Simple Idea: Programs think they have tons of memory. The OS manages what's actually in RAM vs. on disk.

Real-world example: Your computer has 8GB RAM but can run programs needing 100GB total. How? Some data is on the hard drive.

4.2 Demand Paging

What is it? Only load data into RAM when the program needs it.

How it works: 
1. Program accesses memory location
2. Location not in RAM → Page Fault!
3. OS loads page from disk to RAM
4. Program continues

4.3 Page Replacement Algorithms

The Problem: RAM is full. Need to load a new page. Which old page should we remove?

Three Popular Algorithms:
  • FIFO (First-In-First-Out): Remove the oldest page. Simple but not smart.
  • LRU (Least Recently Used): Remove the page not used for longest time. Better!
  • LFU (Least Frequently Used): Remove the page used least often. Also good!

4.4 Performance of Virtual Memory

Formula: Effective Access Time (EAT)

EAT = (1 - p) × m + p × (d + m)

p = page fault rate (0 to 1)
m = memory access time (fast, ~100 ns)
d = disk access time (slow, ~8 ms)

Example: If p = 0.001 (0.1% page faults)
EAT = 0.999 × 100 + 0.001 × 8100 = 107 ns

4.5 Thrashing

What is it? System spends more time swapping pages than actually running programs.

Real-world example: Your computer is constantly using the hard drive, programs run very slowly.

Causes:
  • Too many programs running
  • Not enough RAM
  • Poor page replacement algorithm

4.6 Copy-on-Write

What is it? When a program creates a copy, don't actually copy data yet. Only copy when data is modified.

Real-world example: Making a photocopy. Don't copy everything - only copy the pages someone changes.

4.7 Non-Uniform Memory Access (NUMA)

What is it? In multi-processor systems, some memory is closer to some processors.

Real-world example: In an apartment building, the mailbox near your door is faster to access than one far away.
